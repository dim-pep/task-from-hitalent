# QnA API сервис

Небольшой API‑сервис для вопросов и ответов (Questions / Answers), реализованный в рамках тестового задания.

## Как запустить
Склонировать репозиторий к себе
```bash
git clone https://github.com/dim-pep/task-from-hitalent.git
cd task-from-hitalent
```
Создать .env командой 
```bash
cat .env.example >> .env
```
при желании значение переменных можно поменять  

Запуск через докер из корневой папки проекта
```
docker compose up --build
```

Будут подняты:

- db-1 = PostgreSQL  
- migrations-1 = контейнер с goose, который применит миграции  
- api-1 = Go‑приложение

API будет доступно по адресу:
http://localhost:8080  
Адрес может быть другой если вы меняли значение переменной APP_PORT  

## Выполнение технических требований

### Использование net/http

- Весь HTTP‑слой реализован на стандартной библиотеке `net/http`
- Весь код отвечающий за это находится в `internal/web`
- Для сериализации/десериализации используется `encoding/json`

### GORM

- Всё взаимодействия с БД, кроме миграций, реализованно через библеотеку `GORM` 
- Весь код отвечающий за это находится в `internal/db`

### PostgreSQL

- В docker-compose.yml поднимается PostgreSQL
- Настроен volume pg_data

### goose

- SQL‑миграции хранятся в директории `migrations/`
- В docker-compose.yml поднимается сервис, который прогоняет миграции при старте

### Docker

- Реализован Dockerfile для api находится в корне проекта
- Реализован docker-compose.yml со всеми сервисами, находится там же.

### Логирование 

- Использует стандарнтый пакет `log`

### Испозльзование httptest,testify

Реализован 1 интерграционный тест находится по пути `internal/web/question_test.go`  
Запустить можно из корневой папки проекта при запущенной бд  
Команда: `go test ./internal/web`
## Postman  
Я тестировал работу api через Postman, и оставил конфиг моих запросов ко всем эндпоинтам на случай если для тестирования тоже будете использовать Postman.  
Файл конфига `hitalent.postman_collection.json` находится в корневой папке проекта
